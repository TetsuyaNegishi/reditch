# Reditch

スクリーンショットに矢印・文字・図形を描き込んだり、機密部分をモザイクで隠したりできる画像注釈ツールです。もとはブラウザだけで動くWebアプリとして作られ、現在はElectronによるmacOSデスクトップアプリとして配布されています（Webブラウザ版も引き続き利用可能）。

- ビルド不要・外部依存ゼロ（Vanilla JS + HTML5 Canvas、Electronはデスクトップ配布のためのラッパー）
- 完全ローカル動作（画像データは外部送信されません）
- デスクトップ版では範囲指定スクリーンショット・ドラッグアウト共有などのOS統合機能を利用可能

## インストール（macOSアプリ）

Homebrewで導入できます（Apple Silicon Mac向け）。

```bash
brew install --cask tetsuyanegishi/reditch/reditch
```

または[Releasesページ](https://github.com/TetsuyaNegishi/reditch/releases)から `.dmg` をダウンロードして手動でインストールすることもできます。

> **Note**
> 現時点ではApple Developer IDによるコード署名・公証を行っていない未署名ビルドです。初回起動時にGatekeeperの警告が出る場合があります。

## 起動方法（デスクトップアプリ・開発用）

```bash
npm install
npm start
```

macOSアプリとしてウィンドウが起動します。

### 配布用パッケージの作成

```bash
npm run dist   # electron-builder --mac : .dmg / .zip を dist/ に生成
npm run pack   # electron-builder --dir : dist/mac 配下に .app を生成（署名・圧縮なしの動作確認用）
```

`v*.*.*` タグをpushするとGitHub Actionsが自動でビルド・GitHub Releaseへの公開まで行う。手順は[docs/RELEASE.md](docs/RELEASE.md)を参照。

## Webブラウザで使う

ビルド不要です。ローカルHTTPサーバーでプロジェクトルートを配信して開いてください。

```bash
python3 -m http.server 8000
```

ブラウザで `http://localhost:8000` を開きます。

> **Note**
> - ES Modules（`<script type="module">`）を使っているため、`file://` で直接 `index.html` を開いても動作しません。必ずHTTPサーバー経由でアクセスしてください。
> - クリップボードへのコピー（`navigator.clipboard`）は `localhost` またはHTTPS環境でのみ動作します。それ以外の環境ではコピーが失敗するため、代わりにPNG保存を利用してください。
> - Web版には範囲指定スクリーンショットやドラッグアウト共有などのOS統合機能はありません（後述のデスクトップ限定機能を参照）。

## 使い方

### 画像を読み込む

以下のいずれかの方法で画像を読み込めます。

1. ツールバーの「開く」ボタン、または初期画面の「ファイルを選択」からファイル選択ダイアログで選ぶ
2. 画像ファイルをウィンドウにドラッグ&ドロップする
3. クリップボードにコピーされた画像を `⌘V` / `Ctrl+V` で貼り付ける

大きな画像はウィンドウに収まるよう縮小表示されますが、内部解像度（保存時の実解像度）は維持されます。

### 注釈ツール

ツールバーから選択、またはショートカットキーで切り替えます。

| ツール | 説明 |
| --- | --- |
| 選択・移動 | 注釈をクリックで選択し、ドラッグで移動。ハンドルでリサイズ・回転、`Delete`/`Backspace`で削除 |
| 矢印 | ドラッグした方向に矢印を描画 |
| 直線 | ドラッグで直線を描画 |
| 四角形 | ドラッグで矩形（枠線）を描画 |
| 楕円 | ドラッグで楕円（枠線）を描画 |
| ペン（フリーハンド） | ドラッグした軌跡に沿って自由曲線を描画 |
| テキスト | クリックした位置にテキストボックスが表示され、文字を入力・確定 |
| モザイク | ドラッグで指定した矩形領域をピクセレート処理し判読不能にする |

### 色・線の太さ・文字サイズ

- ツールバーのカラースウォッチ（9色パレット）から描画色を選択
- 線の太さ（細・中・太・極太）、文字サイズ（小・中・大・特大）をセレクトボックスで変更
- 注釈を選択中に色・線幅・文字サイズを変更すると、その注釈自体にも反映されます

### 選択・移動・削除

「選択・移動」ツールで注釈をクリックすると選択状態になり、そのままドラッグで移動できます。`Delete` または `Backspace` キーで選択中の注釈を削除、`Escape` キーで選択解除します。

選択中の注釈にはハンドルが表示され、サイズや向きを変更できます。

- 矢印・直線: 両端のハンドルで長さ・向きを変更
- 四角形・楕円: 4隅のハンドルでリサイズ、枠の上に伸びた青いハンドルで回転
- モザイク: 4隅のハンドルでリサイズ
- ペン: 4隅のハンドルで拡大縮小
- テキスト: 右下のハンドルで文字サイズを変更

### Undo / Redo

`⌘Z` / `Ctrl+Z` で元に戻す、`⇧⌘Z` / `Ctrl+Shift+Z` でやり直しができます。すべての描画・移動・削除・プロパティ変更操作が対象です。

### 保存・コピー

- 「保存」ボタン: 注釈・モザイクを合成した画像をPNGファイルとしてダウンロード
- 「コピー」ボタン: 合成した画像をクリップボードへコピーし、他アプリへ貼り付け可能にする
- デスクトップ版ではクリップボードコピーはOSネイティブ（Electronのipcを介した`clipboard` API）で行われ、Web版の`navigator.clipboard`より安定して動作します

## デスクトップ限定機能

デスクトップアプリ（`npm start`で起動するElectron版）でのみ利用できる機能です。

### 範囲指定スクリーンショット

- `⌘⇧7`（グローバルショートカット。アプリが背面にあっても有効）またはツールバーの「キャプチャ」ボタン、ファイルメニューの「範囲を指定してキャプチャ」から起動
- macOSの`screencapture`コマンドを使った対話的な範囲選択が始まり、選択した範囲を撮影します
- 撮影が完了すると自動的にアプリがフォアグラウンドに戻り、キャプチャ画像がそのまま編集画面に読み込まれます
- `Esc`キーでキャプチャをキャンセルできます（この場合は画像読み込みは行われません）
- macOSのみ対応の機能です（他OSでは動作しません）
- 初回実行時、macOSの「画面収録」権限の許可を求められる場合があります。許可されない場合は「システム設定 > プライバシーとセキュリティ > 画面収録」からReditchの権限を有効にしてください

### ドラッグアウト共有

ツールバーの「ドラッグで共有」ハンドルをFinderやSlackなど他のアプリへドラッグすると、現在の編集内容（注釈・モザイクを合成済み）をPNGファイルとしてそのアプリへ渡せます。

### メニュー・ショートカット

| 操作 | ショートカット |
| --- | --- |
| 範囲を指定してキャプチャ | `⌘⇧7` |
| 画像を開く | `⌘O` |
| PNGとして保存 | `⌘S` |
| 画像をコピー | `⇧⌘C` |

これらはファイルメニューからも実行できます。

## キーボードショートカット

| キー | 動作 |
| --- | --- |
| `V` | 選択・移動ツール |
| `A` | 矢印ツール |
| `L` | 直線ツール |
| `R` | 四角形ツール |
| `O` | 楕円ツール |
| `P` | ペンツール |
| `T` | テキストツール |
| `M` | モザイクツール |
| `⌘Z` / `Ctrl+Z` | Undo（元に戻す） |
| `⇧⌘Z` / `Ctrl+Shift+Z` | Redo（やり直す） |
| `Delete` / `Backspace` | 選択中の注釈を削除 |
| `Escape` | 選択解除 |
| `⌘V` / `Ctrl+V` | クリップボードから画像を貼り付け |

テキスト編集中はこれらのショートカットは無効になります。

## アーキテクチャ概要

コア部分（画像編集ロジック）はビルド不要のクライアントサイドSPAです。Vanilla JS（ES Modules）と HTML5 Canvas のみで構成され、依存ライブラリはありません。このコアをElectronでラップすることでデスクトップアプリ化しており、Web版・デスクトップ版は同一の`index.html`/`css`/`js`を共有します。

注釈は「ベクターデータ（itemオブジェクトの配列）」として状態に保持し、毎フレーム `ベース画像 → モザイク → 図形/文字` の順にCanvasへ再描画する方式を採っています。この方式により、選択・移動・Undo/Redoを自然に実現しています。

デスクトップ版では、Electronのメインプロセス（`electron/main.cjs`）がメニュー・グローバルショートカット・範囲キャプチャ（`screencapture`起動）・クリップボード・ドラッグアウトといったOS統合機能を担当し、`preload.cjs`経由で`window.electronAPI`としてレンダラー（`js/main.js`）に公開します。`js/main.js`は`window.electronAPI`の有無でWeb版とデスクトップ版の分岐を行います。

### ディレクトリ構造

```
reditch/
├── index.html
├── css/style.css
├── js/
│   ├── main.js        # 起動・イベント配線・キーボードショートカット・Electron連携
│   ├── state.js        # 状態管理 + undo/redo
│   ├── renderer.js     # 再描画・モザイク合成・書き出し用合成
│   ├── io.js            # 画像の読み込み/保存/クリップボード
│   └── tools/
│       ├── registry.js  # ツールのレジストリ（type → モジュール）
│       ├── arrow.js / line.js / rect.js / ellipse.js / pen.js
│       ├── text.js / mosaic.js / select.js
│       └── geom.js      # 図形計算の共通ユーティリティ
├── electron/
│   ├── main.cjs         # メインプロセス（メニュー・グローバルショートカット・範囲キャプチャ・クリップボード/ドラッグアウトIPC）
│   └── preload.cjs       # window.electronAPIの公開（contextIsolation経由）
├── package.json          # scripts: start / dist / pack、electron-builder設定
└── README.md
```

各ツールは同一インターフェース（`onDown` / `onMove` / `onUp`、および `draw` / `hitTest`）を実装し、`tools/registry.js` がレジストリとして管理します。

## スコープ外・既知の制限

以下は現時点で未対応です。

- 画像のクロップ（切り抜き）
- 範囲指定スクリーンショットはデスクトップ版のみ対応（Web版では利用不可）、かつmacOSのみ対応（Windows/Linuxでは動作しません）
- プロジェクトの保存・再編集（書き出しはPNGへの焼き込みのみで、注釈データは保持されません）
- モバイル・タッチ操作の最適化
